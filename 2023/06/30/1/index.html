聚合函数

函数说明
AVG()返回某列的平均值
COUNT()返回某列的行数
MAX()返回某列的最大值
MIN()返回某列的最小值
SUM()返回某列值之和
STD()返回某列的标准差

以上三种方式效率相同, 但是 Count(1) 会更快一些, 因为它不会检查表中的列名或者表达式是否存在.
count(*) 会检查列名是否存在, 但是不会检查表达式是否存在.
count(列名) 会检查列名是否存在, 但是不会检查表达式是否存在.

SELECT 小结

SELECT 是 SQL 中最重要的命令, 也是最复杂的命令.

[] 表示可选, {} 表示必选, | 表示可选其中之一.
xxx AS xxx 表示别名, 用于查询结果中的字段名.
* 表示所有字段, table.* 表示指定表的所有字段.
xxx join xxx 表示两个表的连接方式, 一般有 left join, right join, inner join.
WHERE 表示条件, 用于筛选数据.
GROUP BY 表示分组, 用于分组统计.
HAVING 表示筛选, 用于筛选分组后的数据, 与 WHERE 的区别是, WHERE 是在分组前筛选, HAVING 是在分组后筛选.
ORDER BY 表示排序, 用于排序数据.
LIMIT 表示分页, 用于分页查询.

事务

事务是由一组 SQL 语句组成的逻辑处理单元, 事务具有以下四个特性, 通常称为事务的 ACID 特性.
要么全部执行, 要么全部不执行.

  • 原子性(Atomicity): 事务是一个原子操作单元, 其对数据的修改, 要么全都执行, 要么全都不执行.

  • 一致性(Consistency): 在事务开始和完成时, 数据都必须保持一致状态.

  • 隔离性(Isolation): 数据库系统提供一定的隔离机制, 保证事务在不受外部并发操作影响的"独立"环境执行.

  • 持久性(Durability): 事务完成之后, 它对于数据的修改是永久性的, 即使出现系统故障也能够保持.

eg: 转账

隔离导致的问题: 脏读, 不可重复读, 幻读.

脏读: 一个事务读取到了另一个事务未提交的数据.
不可重复读: 一个事务读取到了另一个事务已提交的 update 数据.
幻读: 一个事务读取到了另一个事务已提交的 insert 数据.

事务的隔离级别

  • 读未提交(Read Uncommitted): 一个事务还没提交时, 它做的变更就能被别的事务看到.

  • 读提交(Read Committed): 一个事务提交之后, 它做的变更才会被其他事务看到.

  • 可重复读(Repeatable Read): 一个事务执行过程中看到的数据, 总是跟这个事务启动时看到的数据是一致的.

  • 串行化(Serializable): 对于同一行记录, 读写会加锁, 事务只能一个接一个地执行, 不能并发执行.

事务实现转账

MYSQL 是默认事务自动提交的, 也就是说, 每执行一条 SQL 语句, 就会自动提交一次事务.
但是我们也可以手动处理事务，使用语句 SET AUTOCOMMIT = 0; 关闭自动提交事务, 然后使用 COMMIT 提交事务, 使用 ROLLBACK 回滚事务.
