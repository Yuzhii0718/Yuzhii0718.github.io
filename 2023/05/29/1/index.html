异常

异常的概念

异常是程序在执行过程中出现的错误，异常是程序运行时出现的错误，与语法错误不同，异常不会在编译时出现，而是在运行时出现。

异常的分类：

  • Error：错误，是程序无法处理的错误，表示运行应用程序中较严重问题，如系统崩溃、虚拟机错误、内存溢出等，这些异常发生时，Java虚拟机（JVM）一般会选择线程终止。

  • Exception：异常，是程序本身可以处理的异常，表示程序运行过程中出现的问题，可以使用 try…catch…finally 语句进行处理。

异常的处理

异常处理的方式有两种：

  • 使用 try…catch…finally 语句进行处理

  • 使用 throws 关键字声明异常，将异常抛给调用者进行处理

try…catch…finally 语句

try…catch…finally 语句用于捕获异常，try 语句块用于捕获异常，catch 语句块用于处理异常，finally 语句块用于释放资源。

throws 关键字

throws 关键字用于声明异常，将异常抛给调用者进行处理。

throws 与 throw 的区别：throws 用于声明异常，throw 用于抛出异常。但是 throws 与 throw 的使用方式相似，都是后面跟异常类的对象。

自定义异常

自定义异常类需要继承 Exception 类或者 RuntimeException 类。

异常的处理总结

  • 采用逻辑去合理规避异常，同时辅助 try…catch…finally 语句进行异常处理

  • 多重 catch 语句块，从小到大进行捕获，避免大范围的异常捕获，用catch(Exception)处理可能会遗漏异常

  • 对于不确定的代码，可以使用 try…catch…finally 语句进行异常处理

  • 尽量处理异常，不要直接printStackTrace()，可以使用日志框架进行日志记录

  • finally 语句块中的代码一定会被执行，除非在 finally 语句块中使用了 System.exit(0) 退出程序
