变量，常量与作用域

变量

变量是程序中最基本的存储单元，其值可以在程序运行过程中发生改变。Java 中的变量分为以下几种：

  • 局部变量

  • 类变量（静态变量）

  • 成员变量（非静态变量）

局部变量：在方法、构造方法或语句块中定义的变量称为局部变量。局部变量在方法、构造方法、语句块执行时创建，当它们执行完成后，变量将会被销毁。访问修饰符不能用于局部变量。

类变量（静态变量）：在类中以 static 关键字声明，但必须在方法、构造方法和语句块之外。无论一个类创建了多少个对象，类只拥有类变量的一份拷贝。静态变量除了被声明为常量外很少使用，因为它们的值在程序运行期间不能改变。

除了基本类型，变量的默认值为 null。

成员变量（非静态变量）：在类中以 public、protected 或 private 来声明的变量称为成员变量。如果变量使用 final 关键字声明，那么它就是一个常量，常量在声明时必须被赋值，赋值后值不再改变。

常量

常量在程序运行时不能被修改的量。在 Java 中使用 final 关键字来修饰常量，声明方式和变量类似，但是常量声明时必须被赋值，赋值后值不再改变。

常量在声明时必须被赋值，赋值后值不再改变。
常量名通常使用大写字母。

static 关键字用来声明独立于对象的静态变量，无论一个类实例化多少对象，它的静态变量只有一份拷贝。static 关键字用来声明独立于对象的静态方法。
final 关键字用来修饰类、方法和变量，final 修饰的类不能被继承，修饰的方法不能被重写，修饰的变量为常量，是不可修改的。

变量的命名规则

  • 变量的命名可以包含字母、数字、下划线和美元符号，但变量名不能以数字开头。

  • 合法的变量名示例：age、$salary、_value、__1_value。

  • 见名知意：变量，类名，方法名应该尽可能短，同时要能够体现变量的作用。

  • 大小写敏感：age 和 Age 是两个不同的变量。

  • 驼峰命名法（局部变量和类成员变量）：firstName、lastName、myAge、mySalary。

  • 类名：首字母大写，如 MyClass。

  • 方法名：首字母小写，如 myMethod()。

位运算符

Java 定义了位运算符，应用于整数类型（byte、short、int 和 long）的变量。位运算符作用在所有的位上，并且按位运算。假设 A 的值为 60，B 的值为 13，它们的二进制格式表示将如下：

位运算符作用于位，并逐位执行操作。&、| 和 ^ 的真值表如下所示：

运算符描述实例
&如果相对应位都是 1，则结果为 1，否则为 0。(A & B) 将得到 12，即为 0000 1100
|如果相对应位都是 0，则结果为 0，否则为 1。(A | B) 将得到 61，即为 0011 1101
^如果相对应位值相同，则结果为 0，否则为 1。(A ^ B) 将得到 49，即为 0011 0001
~按位取反运算符翻转操作数的每一位，即 0 变成 1，1 变成 0。(~A ) 将得到 -61，即为 1100 0011，一个有符号二进制数的补码形式。
<<按位左移运算符。左操作数按位左移右操作数指定的位数。A << 2 将得到 240，即为 1111 0000
>>按位右移运算符。左操作数按位右移右操作数指定的位数。A >> 2 将得到 15，即为 1111
>>>按位右移补零操作符。左操作数的值按右操作数指定的位数右移，移动得到的空位以零填充。A >>> 2 将得到 15，即为 0000 1111

左移运算符 << 的结果是把操作数的所有位都左移指定的次数。左移 n 位就相当于乘以 2 的 n 次方。右移运算符 >> 是 << 运算符的逆运算。右移 n 位就相当于除以 2 的 n 次方。位移运算符 >>> 是 >> 运算符的逆运算，但它使用的是 0 来填充左侧空位。

包

包（package）用于管理 Java 类，类似于文件夹，用于管理文件。包名与文件夹名相同，包名使用小写字母，文件夹名使用小写字母或下划线。包名的命名规范为：com.companyname.project.module。

JavaDoc

JavaDoc 是一个 Java 文档生成工具，可以从源代码中的文档注释中抽取出有用的信息，并生成一个 HTML 文档。JavaDoc 生成的 HTML 文档包含了类、方法、字段等的描述，以及相关的注释。

JavaDoc 标签
JavaDoc 标签是以 @ 开头的，如 @param、@return 等。常见的 JavaDoc 标签如下所示：

  • @param：说明方法的参数。

  • @return：说明方法的返回值。

  • @exception：说明方法可能抛出的异常。

  • @version：说明类的版本。

  • @author：作者

  • @since：从哪个版本开始有的。

  • @see：参考转向，也就是相关主题。

  • @deprecated：说明类、方法已经不建议使用。

  • @serial：说明一个序列化属性。

  • @throws：和 @exception 相同。
