Java 方法

什么是方法

方法是将具有独立功能的代码块组织成为一个整体，使其具有特殊功能的代码集。方法是一种功能的抽象，它们用来解决一类问题，而不是解决某个特定的问题。

方法的定义格式

  • 修饰符：public static

  • 返回值类型：方法最终产生的数据结果的数据类型

  • 方法名：符合命名规则即可

  • 参数类型：产生数据结果需要的未知数据，可以有多个，也可以没有

  • 参数名：参数变量名，符合命名规则即可

  • 方法体语句：完成功能的代码

  • return：结束方法，将返回值带给调用者

  • 返回值：将方法执行后的结果带给调用者

方法的注意事项

  • 方法应该定义在类中，但不能在方法中再定义方法

  • 方法定义的前后顺序无所谓

  • 方法定义之后不会执行，如果希望执行，一定要调用：单独调用、赋值调用、打印调用

方法的重载

在同一个类中，允许存在一个以上的同名方法，只要它们的参数个数或者参数类型不同即可。

通过方法重载，可以让功能更加强大，但是，调用起来更加简单。

方法重载规则

  • 方法名必须相同

  • 参数列表必须不同

    • 个数不同
    • 类型不同
    • 顺序不同

  • 方法的返回值类型可以相同也可以不同

  • 仅仅返回值类型不同，不足以成为方法的重载

方法的递归调用

方法自己调用自己，被称为方法的递归调用。

递归调用虽然有点疯狂，但是递归调用解决问题非常的简洁。

递归调用的使用前提：当调用方法的时候，方法的主体不变，每次调用方法的参数不同，可以使用递归调用。

递归有可能导致栈内存溢出，必须防止递归的次数过多。因此，递归有两个限定条件：

  • 递归的次数不能太多

  • 递归的目的要明确

在递归的结构中，有两个部分：

  • 递归头：递归的结束条件，不再调用方法，返回的是一个固定值

  • 递归体：方法继续调用，计算更复杂的结果，每次调用方法的参数不同
