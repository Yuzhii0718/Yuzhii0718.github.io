日期和时间

类型名称大小
DATE日期3 字节
TIME时间3 字节
DATETIME日期和时间8 字节
TIMESTAMP时间戳4 字节
YEAR年份1 字节

时间格式：YYYY-MM-DD HH:MM:SS时间戳格式：YYYYMMDDHHMMSS

时间戳是一个特殊的时间格式，它是一个整数，可以用来表示一个时间，时间戳的范围是 1970-01-01 00:00:01 到 2038-01-19 03:14:07，超出这个范围就会出现问题。

null 和 not null
null 表示空值，not null 表示非空值，null 和 not null 是列的属性，可以用来限制列的值是否可以为空。

不要把 null 和 0 混淆，null 表示空值，0 表示数字 0。
不要用NULL进行运算

数据库的字段属性

属性描述说明
AUTO_INCREMENT自动增长用于自动增长列的值
DEFAULT默认值用于设置列的默认值
PRIMARY KEY主键用于设置主键
NOT NULL非空用于设置列的值是否可以为空
UNIQUE唯一用于设置列的值是否唯一
COMMENT注释用于设置列的注释

UNSIGNED无符号用于设置列的值是否可以为负数
ZEROFILL零填充用于设置列的值是否用 0 填充

每一个表必须要有以下几个字段：
id 主键
version 乐观锁
is_deleted 软删除
gmt_create 创建时间
gmt_modified 修改时间

引擎(Engine)

引擎是数据库的核心，它决定了数据库的性能、事务、锁、崩溃后的安全恢复等，MySQL 支持多种引擎，常用的引擎有 InnoDB 和 MyISAM。

引擎事务外键锁崩溃后的安全恢复全文索引
InnoDB支持支持行级锁支持不支持
MyISAM不支持不支持表级锁不支持支持

  • MyISAM 节约空间，性能快

  • InnoDB 安全性高，支持事务，支持外键，支持崩溃后的安全恢复

数据库的本质是文件，每个表都是一个文件，每个数据库都是一个文件夹，数据库的引擎就是用来管理这些文件的。

  • .frm 文件：表结构

  • .ibd 文件：表数据和索引
