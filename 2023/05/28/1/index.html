面向对象·四 - Object Oriented Part IV

instanceof 与 类型转换

instanceof 运算符用于判断一个引用类型变量所引用的对象是否是一个类（或子类）的实例，即是否继承于该类（或子类）。

简单来说，instanceof 的作用就是判断一个对象是否是一个类的实例，或者判断一个对象是否是该类及其子类的实例。instanceof 的判断原理是通过判断对象是否是该类的实例，或者判断对象是否是该类父类的实例，或者判断对象是否是该类父类的父类的实例，直到判断到该类的父类为止。

在进行强制类型转换之前，先使用 instanceof 运算符进行判断，避免 ClassCastException 异常的发生。

在强制转换后，可能丢失精度，子类转换为父类不会丢失精度，父类转换为子类会丢失精度。

static 修饰符

static 修饰符用来修饰成员变量和成员方法，static 修饰的成员变量称为静态变量，static 修饰的成员方法称为静态方法。

静态的成员变量和成员方法都可以直接通过类名来访问，不需要创建类的实例，但是静态的成员方法只能访问静态的成员变量和静态的成员方法，不能访问非静态的成员变量和成员方法。

多线程中，静态变量和静态方法是多个线程共享的，非静态变量和非静态方法是多个线程各自拥有的。

静态代码块是在类加载时执行的，而且只执行一次，通常用来初始化静态变量。

生命周期：静态变量和静态方法在类加载时创建，类卸载时销毁，生命周期最长，实例变量和实例方法在类实例化时创建，类实例化时销毁，生命周期中等，局部变量和局部方法在方法调用时创建，方法调用结束时销毁，生命周期最短。

执行优先级：静态初始化块 > 静态变量 > 静态方法 > 实例变量 > 实例方法 > 局部变量 > 局部方法

静态导入：使用 import static 语句可以导入一个类的静态变量和静态方法，导入后可以直接使用静态变量和静态方法，而不需要使用类名来访问。

final 修饰符：final 修饰的类不能被继承，final 修饰的方法不能被重写，final 修饰的变量不能被修改，final 修饰的引用不能指向其他对象，但是可以修改引用指向的对象的内容。

抽象类

抽象类是用 abstract 修饰的类，抽象类不能被实例化，抽象类中可以包含抽象方法和非抽象方法，抽象类的子类必须实现抽象类中的所有抽象方法，否则子类也必须是抽象类。

抽象类只有在被继承时才有意义，抽象类的子类必须实现抽象类中的所有抽象方法，否则子类也必须是抽象类。

抽象类的本质是一个类，所以抽象类可以有构造方法，抽象类的构造方法在子类实例化时会被调用，抽象类的构造方法不能用 abstract 修饰，因为抽象类的构造方法不能被重写，抽象类的构造方法不能用 private 修饰，因为抽象类的构造方法必须被子类调用。

接口 与 抽象类 的区别：

  • 抽象类是用 abstract 修饰的类，抽象类不能被实例化，抽象类中可以包含抽象方法和非抽象方法，抽象类的子类必须实现抽象类中的所有抽象方法，否则子类也必须是抽象类。

  • 接口是用 interface 修饰的类，接口不能被实例化，接口中只能包含抽象方法，接口的子类必须实现接口中的所有抽象方法，否则子类也必须是抽象类。

接口

接口(interface)是用 interface 修饰的类，接口中只能包含抽象方法，接口的子类必须实现接口中的所有抽象方法，否则子类也必须是抽象类。

  • 普通类： 只有具体实现

  • 抽象类： 既有具体实现，又有部分抽象方法

  • 接口： 只有抽象方法

接口就是规范，接口中的所有方法都是抽象方法，接口中的所有变量都是常量，接口中的所有方法都是 public 的，接口中的所有变量都是 public static final 的。

接口的本质是契约，接口是一种特殊的抽象类，接口是一种特殊的类，接口是一种特殊的数据类型。

利用接口，可以变相实现多继承，一个类可以实现多个接口，一个接口可以继承多个接口。

接口的实现类必须实现接口中的所有抽象方法，否则实现类也必须是抽象类。

内部类

（选）内部类是定义在另一个类中的类，内部类可以访问外部类的所有成员，外部类不能访问内部类的成员。

内部类的分类：

  • 成员内部类：定义在类中方法外的类，成员内部类可以访问外部类的所有成员，外部类不能访问内部类的成员。

  • 局部内部类：定义在方法中的类，局部内部类只能在方法中使用，局部内部类可以访问外部类的所有成员，外部类不能访问局部内部类的成员。

  • 匿名内部类：没有名字的内部类，匿名内部类只能在方法中使用，匿名内部类可以访问外部类的所有成员，外部类不能访问匿名内部类的成员。
