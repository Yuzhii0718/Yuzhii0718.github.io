面向对象·三 - Object Oriented Part III

Super

super 关键字，可以用来访问父类内容，而 this 关键字，可以用来访问本类内容。用法也有三种：

  • 1.在本类的成员方法中，访问本类的成员变量。

  • 2.在本类的成员方法中，访问本类的另一个成员方法。

  • 3.在本类的构造方法中，访问本类的另一个构造方法。

super:

  1. 在子类的成员方法中，访问父类的成员变量。

  2. 在子类的成员方法中，访问父类的另一个成员方法。

  3. 和 this 一样，super 也不能出现在静态方法中。并且 super 和 this 不能同时出现在一个方法中。

this: 本身就代表当前对象。所以谁调用，就代表谁。
super: 代表父类对象。所以谁继承，就代表谁。

方法重写

方法重写：子类中出现了和父类一模一样的方法声明。方法重载：本类中出现的方法名一样，参数列表不同的方法。

重写的注意事项：

  1. 必须保证父子类之间方法的名称相同，参数列表也相同。
    @Override：写在方法前面，用来检测是不是有效的正确覆盖重写。这个注解就算不写，只要满足要求，也是正确的方法重写。

  2. 子类方法的返回值必须小于等于父类方法的返回值范围。小扩展提示：java.lang.Object 类是所有类的公共最高父类（祖宗类），java.lang.String 就是 Object 的子类。

  3. 子类方法的权限必须大于等于父类方法的权限修饰符。小扩展提示：public > protected > (default) > private
    备注：(default) 不是关键字 default，而是什么都不写，留空。

  4. 子类方法的异常必须小于等于父类方法的异常范围。小扩展提示：异常的讲解，稍后学习。

为什么要重写？

  1. 当子类需要父类的功能，而功能主体子类有自己特有的内容时，可以重写父类中的方法，这样，即沿袭了父类的功能，又定义了子类特有的内容。

  2. 定义的类的时候，子类的方法要和父类的方法名一样，但是有时候，子类虽然需要父类的功能，但是却不需要父类的方法。这个时候，就可以将父类的方法，覆盖重写。

多态

多态：可以理解为事物存在的多种体现形态。在程序中：

  • 父类引用指向子类对象。

  • 接口引用指向实现类对象。

多态是方法的多态，属性没有多态。

分析：在以上代码中，Animal animal = new Cat(); 这句话，animal 是父类的引用，指向了子类的对象。这就是多态。

多态的前提和体现：

  1. 有继承/实现关系。

  2. 有方法重写。

  3. 有父类引用指向子类对象。

多态的好处：

  1. 提高了代码的维护性（继承保证）

  2. 提高了代码的扩展性（由多态保证）

  3. 多态的出现，是程序设计的一个重要目标。

注意事项：

  1. 多态中，成员方法的访问规则是：看new 的是谁，就优先用谁，没有则向上找。

  2. 多态中，成员变量的访问规则是：无论编译和运行，都看左边（引用型变量）。

  3. 多态中，静态成员方法的访问规则是：无论编译和运行，都看左边。

不能重写的方法：

  1. final 方法。

  2. 静态方法。

  3. private 等子类中不可见方法。
