面向对象·一 - Object Oriented Part I

面向对象是一种思想，是一种设计思想，是一种编程思想。面向对象编程的本质是将复杂的事情简单化，将程序员从执行者转换为指挥者。以类的方式组织代码，以对象的方式组织数据。

面向对象的三大特征

  • 封装

    隐藏对象的属性和实现细节，仅对外提供公共访问方式。

  • 继承

    使用已存在的类的定义作为基础建立新类的技术，新类的定义可以增加新的数据或新的功能，也可以用父类的功能，但不能选择性地继承父类。

  • 多态

    同一操作作用于不同的对象上面，可以产生不同的解释和不同的执行结果。

区别于面向过程

  • 面向过程：强调的是功能行为，以函数为最小单位，考虑怎么做。

    • 优点：性能比面向对象高，因为类调用时需要实例化，开销比较大，比较消耗资源;比如单片机、嵌入式开发、Linux/Unix等一般采用面向过程开发，性能是最重要的因素。
    • 缺点：没有面向对象易维护、易复用、易扩展

  • 面向对象：强调具备了功能的对象，以类/对象为最小单位，考虑谁来做。

    • 优点：易维护、易复用、易扩展，由于面向对象有封装、继承、多态性的特性，可以设计出低耦合的系统，使系统更加灵活、更加易于维护
    • 缺点：性能比面向过程低

静态方法中不能直接访问非静态变量和非静态方法，但是非静态方法可以直接访问静态变量和静态方法。在main方法中，是静态的，所以只能调用静态方法。

作为拓展

为什么要使用静态方法？

  • 静态方法不依赖于对象，即使没有创建对象也可以调用

  • 静态方法只能访问静态成员变量和静态成员方法

  • 静态方法中不能使用this、super关键字

  • 静态方法中不能直接访问非静态成员，可以通过对象来访问非静态成员

  • 静态方法中不能使用非静态方法，可以通过对象来访问非静态方法

什么时候使用静态方法？什么时候使用非静态方法？

  • 如果方法内部没有访问非静态成员变量，那么该方法可以定义为静态方法

  • 如果方法内部访问了非静态成员变量，那么该方法必须定义为非静态方法

类和对象的创建

构造方法

  • 构造方法的作用

    • 创建对象
    • 初始化对象的属性

  • 构造方法的格式

    • 修饰符 类名(参数列表) {}

  • 构造方法注意事项

    • 如果没有定义构造方法，系统会默认给出无参构造方法
    • 如果定义了构造方法，系统不会再提供默认的无参构造方法
    • 构造方法也是可以重载的

  • 构造方法的重载

    • 一个类中可以定义多个构造方法，多个构造方法之间构成重载关系
    • 构造方法重载的好处：方便创建对象的同时，初始化对象的属性

构造器

  • 构造器是一种特殊的方法，用来初始化对象

  • 构造器的名称必须与类名相同

  • 构造器没有返回值，也不需要void

  • 如果没有定义构造器，系统会默认提供一个无参构造器

  • 构造器可以重载

引用类型

  • 引用类型的变量非常类似于C/C++的指针，所有的引用类型默认值都是null

  • 一个引用变量可以用来引用任何与之兼容的类型

  • 例：父类的引用可以指向子类的对象，但是子类的引用不能指向父类的对象

属性

  • 属性是一个类的组成部分，是一个对象的状态信息

  • 属性也被称为成员变量

  • 属性的定义格式：数据类型 属性名 = 初始值;

  • 属性的访问格式：对象名.属性名
