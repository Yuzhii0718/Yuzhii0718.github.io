本文是学习JavaScript的笔记，主要是学习JavaScript的基础知识，包括JavaScript的内部对象、面向对象编程、JSON、Ajax。

方法

方法是指对象的属性，属性的值是函数，因此方法是对象的函数属性。方法是对象的行为，对象的行为就是对象的方法。方法是对象的行为，对象的行为就是对象的方法。

this

this 是指函数运行时所在的环境对象，this 的值在函数运行时才能确定，因此 this 的值是动态的，而不是静态的。

apply

apply 方法的作用是改变函数的运行环境，它的第一个参数是改变后的运行环境，第二个参数是一个数组，表示函数运行时的参数。

JavaScript 中可以控制this的值，这是一种非常强大的功能，因为this的值可以动态改变，因此函数的行为也可以动态改变。这种特性使得JavaScript语言具有很强的灵活性，但是也使得JavaScript语言的编程难度大大增加，因为程序员必须非常清楚函数的运行环境，才能保证程序的正确运行。

内部对象

标准内部对象是指 ECMAScript 规范定义的对象，它们可以在 ECMAScript 程序的任何地方使用，不需要通过程序创建。标准内部对象的属性和方法可以在全局环境中直接使用，也可以在其他对象的环境中使用。

  • typeof 123; // ‘number’

  • typeof ‘abc’; // ‘string’

  • typeof true; // ‘boolean’

  • typeof undefined; // ‘undefined’

  • typeof Math.abs; // ‘function’

  • typeof null; // ‘object’

  • typeof []; // ‘object’

  • typeof {}; // ‘object’

Date 日期对象

时间戳是指1970年1月1日00:00:00（UTC）至今的毫秒数。时间戳可以精确表示时间，因此时间戳常用来做时间运算，比如，判断某个时间是否在当前时间之前。

JSON

JSON 是一种数据格式，它的出现是为了解决 JavaScript 对象的字符串表示问题。JavaScript 对象的字符串表示有两种方式：一种是 JSON 格式，另一种是 JavaScript 语法表示法。JSON 格式的优点是简单，易于读写，但是缺点是不支持函数，因此只能用于数据交换，而不能用于创建 JavaScript 对象。

JSON 和 JavaScript 对象的区别

  • JSON 只支持数组和对象作为根元素，不支持字符串、数值、布尔值等作为根元素。

  • JSON 的键必须用双引号""括起来。

  • JSON 不支持函数，正则表达式等对象。

  • JSON 不支持注释，JSON 格式的文件必须严格遵守 JSON 语法。

Ajax

Ajax 是 Asynchronous JavaScript and XML 的缩写，意思是异步的 JavaScript 和 XML。Ajax 是一种在无需重新加载整个网页的情况下，能够更新部分网页的技术。Ajax 不是新的编程语言，而是一种使用现有标准的新方法。

面向对象编程

什么是面向对象编程？面向对象编程是一种编程范式，它把对象作为程序的基本单元，一个对象包含了数据和操作数据的函数。
JavaScript, Java, C++等语言都支持面向对象编程，但是支持的程度各不相同，JavaScript只是一种基于原型的面向对象编程语言，它的面向对象编程与传统的面向对象编程有很大区别。

面向对象原型继承

  • 类：模板

  • 对象：实例在传统的面向对象编程语言中，类是对象的模板，而对象是类的实例。但是，JavaScript的面向对象编程并不是基于类的，而是基于原型的。JavaScript并没有提供对象继承的语法，而是通过原型复制来实现对象继承。

原型原型是指对象的原型对象，每个对象都会从原型继承属性。

obj对象的原型是Object.prototype，它是所有对象的原型，因此obj对象可以继承到Object.prototype的所有属性和方法。

原型链原型链是指对象的原型对象也是对象，因此它也有自己的原型对象，这样一层一层往上，就构成了原型链。
