本文介绍了 JavaScript 的变量作用域。

变量作用域

变量作用是指变量的作用范围，变量的作用范围决定了变量的生命周期。变量的作用域分为全局作用域和局部作用域。全局作用域是指在整个程序中都可以访问的变量，局部作用域是指在函数内部可以访问的变量。

  • 全局作用域

全局作用域是指在整个程序中都可以访问的变量，全局作用域的变量可以在任何地方访问，但是不推荐这样做，因为全局作用域的变量会污染全局作用域，导致变量命名冲突。

  • 局部作用域局部作用域是指在函数内部可以访问的变量，局部作用域的变量只能在函数内部访问，函数外部无法访问。局部作用域的变量在函数执行完毕后会被销毁，因此局部作用域的变量不会污染全局作用域。

闭包

闭包是指函数内部可以访问函数外部的变量，函数外部无法访问函数内部的变量。闭包的作用是可以让函数外部访问函数内部的变量，但是不推荐这样做，因为闭包会导致内存泄漏。

规范

由于所有的全局变量都会绑定到 window 对象上，因此不推荐使用全局变量，如果一定要使用全局变量，那么必须使用 window 对象来访问全局变量，否则会导致全局变量污染全局作用域。

把所有的全局变量都绑定到一个全局变量上，这样就不会污染全局作用域，也不会造成全局变量命名冲突。

局部作用域

局部作用域是指在函数内部可以访问的变量，局部作用域的变量只能在函数内部访问，函数外部无法访问。局部作用域的变量在函数执行完毕后会被销毁，因此局部作用域的变量不会污染全局作用域。

常量 const

const 命令声明的变量是常量，一旦声明，常量的值就不能改变。在ES6之前，JavaScript使用大写变量表示常量。这种做法难以保证变量的值不会被改动，因此ES6引入了 const 命令，声明一个只读的常量。一旦声明，常量的值就不能改变。
