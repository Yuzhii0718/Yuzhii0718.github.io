索引

在 MySQL 中, 索引是一种特殊的文件(InnoDB 数据表上的索引是表空间的一个组成部分), 它们包含着对数据表里所有记录的引用指针.

索引是对数据库表中一列或多列的值进行排序的一种结构, 使用索引可快速访问数据库表中的特定信息.

索引是一个单独的文件, 它们很快, 但是会占用磁盘空间, 并且在插入, 更新和删除数据时需要额外的时间来更新索引.

索引的本质是数据结构, 数据结构是为了提高数据的检索效率而产生的.

索引的分类

  • 普通索引 Key/Index
    (普通索引是最基本的索引, 没有任何限制)

  • 唯一索引 Unique
    (唯一索引的值必须唯一, 但是允许有空值)

  • 主键索引 Primary Key
    (主键索引的值必须唯一, 且不允许有空值)

  • 全文索引 Fulltext
    (全文索引是对一整段文字进行索引, 例如对一篇文章的标题和正文进行索引, 以便用户可以通过搜索词快速找到该文章)

  • 复合索引
    (复合索引是将多个列组合在一起创建索引, 例如将学生表的学号和姓名组合在一起创建索引, 以便用户可以通过学号和姓名快速找到该学生)

  • 空间索引
    (空间索引是对空间数据类型的字段创建索引, 例如对地理坐标点进行索引, 以便用户可以通过地理坐标快速找到该地点)

使用索引

  1. 在创建表时给字段增加索引

  2. 在创建表之后给字段增加索引

索引的优缺点

优点

  • 大大减少了服务器需要扫描的数据量

  • 帮助服务器避免排序和临时表

  • 将随机 I/O 变为顺序 I/O

  • 可以将随机查询变为索引查询

  • 可以加速表和表之间的连接

  • 在使用分组和排序子句进行数据检索时，可以显著减少查询时间

  • 可以通过使用唯一索引或主键来强制表中每一行的唯一性

  • 可以减少查询中使用的锁定时间

缺点

  • 创建和维护索引需要时间成本, 例如在插入数据时, 除了将数据插入到表中, 还需要将数据插入到索引中

  • 索引需要占用物理空间, 例如在创建索引时, 除了需要占用表的物理空间, 还需要占用索引的物理空间

  • 当对表中的数据进行增删改时, 索引也需要进行维护, 例如在对表中的数据进行增删改时, 除了需要对表中的数据进行增删改, 还需要对索引中的数据进行增删改

  • 索引只能加快查询速度, 不能加快更新表的速度, 例如在对表中的数据进行增删改时, 除了需要对表中的数据进行增删改, 还需要对索引中的数据进行增删改

索引原则

  1. 索引的字段必须是经常被查询的字段

  2. 索引不是越多越好, 索引越多, 插入, 更新, 删除的速度就越慢

  3. 不要对经常变动的字段创建索引, 例如: 订单表的订单状态

  4. 不要对数据量过小的表创建索引, 例如: 一些配置表
