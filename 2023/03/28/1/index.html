严格检查模式

严格模式是 ECMAScript 5 引入的一个新的功能。严格模式是为 JavaScript 定义了一种不同的解析与执行模型。严格模式的目的有以下几个：

  • 消除 JavaScript 语法的一些不合理、不严谨之处，减少一些怪异行为;

  • 消除代码运行的一些不安全之处，保证代码运行的安全；

  • 提高编译器效率，增加运行速度；

  • 为未来新版本的 JavaScript 做好铺垫。

  • 严格模式是通过在脚本或函数的头部添加 “use strict”; 来启用的。

  • 严格模式下，ECMAScript 3 中的一些不安全的操作将会抛出错误，比如：

    • 未声明的变量赋值将会抛出错误；
    • 不能使用 with 语句；
    • 不能对只读属性赋值；
    • 不能使用八进制字面量；
    • 不能删除不可删除的属性；
    • 不能删除变量 delete prop，只能删除属性 delete global[prop]；
    • eval 不会在它的外层作用域引入变量；
    • eval 和 arguments 不能被重新赋值；
    • arguments 不会自动反映函数参数的变化；
    • 不能使用 arguments.callee；
    • 不能使用 arguments.caller；
    • 禁止 this 指向全局对象；
    • 不能使用 fn.caller 和 fn.arguments 获取函数调用的堆栈；
    • 增加了保留字（比如 protected、static 和 interface）。

字符串长度

控制台中输入 message.length 也可以得到字符串的长度。也可以打印下标，console.log(message[0])。

在 JavaScript 中，字符串是不可变的，也就是说，字符串一旦创建，它们的值就不能改变。如果需要改变某个变量保存的字符串，那么只能销毁原来的字符串，然后重新保存一个新的字符串。

大小写转换

substring()、substr()、slice()

  • substring()：第一个参数指定字符串的开始位置，第二个参数指定字符串结束位置，但不包括该位置的字符。

  • substr()：第一个参数指定字符串的开始位置，第二个参数指定要返回的字符个数。

  • slice()：第一个参数指定字符串的开始位置，第二个参数指定字符串结束位置，但不包括该位置的字符。

数组类型详解

array可以存储任何类型的数据，包括基本类型和引用类型。数组的长度是动态的，可以随时添加和删除元素。数组的每一项都有自己的索引，索引从0开始。数组的索引是属性名的一种特殊形式，可以使用点表示法或方括号表示法来访问数组的元素。

数组的length属性

对象类型详解

对象是属性的无序集合，每个属性都由一个名字和一个值组成。对象的属性名可以是任意字符串，包括空字符串。属性值可以是任意的数据类型，包括基本类型和引用类型。对象的属性名是字符串，因此可以使用方括号表示法来访问对象的属性。
