枚举

什么是枚举？

枚举是一种用户自定义的数据类型，它是由一组枚举常量组成的，枚举常量的值是从0开始递增的整数。

使用枚举可以提高程序的可读性。让代码更加清晰。以及可以在一定程度上增加程序的健壮性、安全性。

但是一般情况下，我们不会使用枚举，因为它的使用场景比较少。

进制转换

什么是进制？

进制是一种计数的方法，它是一种数学计数方法。在计算机中，常用的进制有二进制、八进制、十进制、十六进制。它们用英文表示分别是：Binary、Octal、Decimal、Hexadecimal。用在数字后的字母缩写表示不同的进制。例如：

  • 二进制：101B

  • 八进制：76Q

  • 十进制：123D

  • 十六进制：A1H

带符号数

  • 原码

    原码就是符号位加上真值的绝对值，即用第一位表示符号，其余位表示值。

    8位二进制数的原码表示范围是：[-127, 127]

  • 补码

    补码就是在原码的基础上，符号位不变，其余位取反，然后加1。

    e.g.

    补码被广泛应用于计算机系统中，因为补码的加法运算比较简单。

  • 反码

    反码就是在原码的基础上，符号位不变，其余位取反。

    反码运算不便，一般不使用。

  • 移码

    移码表示数值平移了一个固定的量 N，这个量的大小等于2^(n-1)，n表示二进制数的位数。N 称为移码的模量。

    移码主要用于浮点数的表示。

补码怎么求？

存在一个问题：如何表示一个负数？例如：-5，这个数肯定与 5 不一样。

我们求十进制转换二进制的时候，是将十进制数不断除以 2，然后将余数倒序排列。但是这个方法对于负数就不适用了，因为负数除以 2 之后，余数就不是 0 或者 1 了。

我们可以将负数的二进制表示成正数的二进制表示，然后在最高位加上符号位，0 表示正数，1 表示负数。

GitHub Copilot: 当补码不够位数时，左边应该补符号位，即正数补0，负数补1。这是因为补码的最高位是符号位，用来表示数的正负。

而负数的补码就是它的反码加1。
